Core interface for managing driver login with Driveri using mobile phone as BLE beacon. Obtain an instance by calling getAppBeaconAPI.

In order to achieve Driver login using mobile phone as BLE beacon, phone's location services must be turned ON and App should have android.Manifest.permission.ACCESS_FINE_LOCATION permission. DriveriSDK will send LOCATION_SERVICES_TURNED_OFF_BROADCAST and LOCATION_PERMISSION_NOT_GRANTED_BROADCAST broadcast if phone's location service is OFF and if App does not have the required location permission respectively.

DriveriSDK will set up AppBeacon feature only if it is enabled for a tenant in IDMS. App can call disableAppBeaconLogin to disable driver login using BLE even if it is enabled for the tenant in IDMS.

DriveriSDK provides logged in driver id to Driveri device as part of App beacon-based driver login feature. Driveri uploads this driver id to Cloud which in turn assigns events generated on the device to this driver id. In case there are multiple drivers in a vehicle, Driveri device will get multiple driver ids and Netradyne Cloud will be unable to identify the actual driver from among a team of drivers. In this case, App can ask drivers to confirm if they have started driving or have stopped driving or they are not driving at this moment. NDAppBeaconAPI has startDriving, stopDriving and markAsNotDriving methods for these cases.
